# `.ai-dev/` (documentación agnóstica del IDE)

Esta carpeta contiene documentación y playbooks pensados para ser reutilizados en cualquier proyecto/IDE.

- `rules/`: estándares normativos (cómo debe ser el código).
- `skills/`: procedimientos/recetas (cómo llegar al estándar: pasos, checklist y validaciones).

Estos documentos se pueden "adaptar" después a herramientas concretas (Cursor rules/skills, AGENTS.md, plantillas de PR, etc.), manteniendo aquí la fuente de verdad.
